# amazing_cc_skills

79 curated Claude Code skills for startups, organized by 8 journey phases and 63 scenarios.

## Project Structure
- `skills/` — 79 skill directories, each with `SKILL.md` + supporting files
- `rankings.json` — **Source of truth** — rankings, scenarios, scores (read/write)
- `skills-manifest.json` — Checksums and metadata (used by installer)
- `SCENARIOS.md` — Auto-generated decision tree for humans (do not edit)
- `RANKINGS.md` — Auto-generated leaderboard for humans (do not edit)
- `install.sh` / `uninstall.sh` / `rank.sh` — CLI tools
- `lib/` — `common.sh`, `manifest.py`, `rank.py`, `scenarios.py`

## How to Query rankings.json

rankings.json has a `_schema` key that describes every field's type and shape. Key query patterns:

```python
import json
r = json.load(open("rankings.json"))

# Find the best skill for a specific problem:
r["scenarios"]["4-get-found"]["scenarios"]["not-ranking-in-google"]["skills"]
# → ["seo-audit"]

# Top 5 skills overall:
r["leaderboard"][:5]
# → [{rank: 1, name: "gstack", score: 100.0, tier: "S", phase: "2-build"}, ...]

# All S-tier skills:
r["by_tier"]["S"]
# → [{name: "gstack", score: 100.0, phase: "2-build"}]

# All skills in the "Ship" phase:
r["by_phase"]["3-ship"]
# → [{name: "ship", score: 25.0, tier: "C"}, ...]

# Check a skill's score and signals:
r["skills"]["ship"]["score"]       # → 25.0
r["skills"]["ship"]["tier"]        # → "C"
r["skills"]["ship"]["signals"]     # → {depth: 0.0, mentions: 0, ...}

# Get all scenarios for a phase:
r["scenarios"]["6-convert"]["scenarios"]
# → {scenario_id: {trigger: "...", problem: "...", skills: [...]}}

# Search scenarios by keyword (CLI):
# python3 lib/scenarios.py --search "bug"
```

### Writable fields
Only `user_overrides` should be written by LLMs. All other fields are regenerated by `rank.sh`.

```python
# Override a skill's tier manually:
r["user_overrides"]["my-skill"] = {"score": 90.0, "tier": "S"}
# This survives re-ranking.
```

## Data Flow
```
rank.sh → lib/rank.py → rankings.json (source of truth)
                              ↓
                        SCENARIOS.md (human: decision tree)
                        RANKINGS.md  (human: leaderboard)
                        skills-manifest.json (installer: checksums)
```

## Scripts
- `./install.sh` — `--copy`, `--select`, `--jobs N`, `--offline`, `--update`, `--dry-run`
- `./rank.sh` — `--offline`, `--json`
- `python3 lib/manifest.py` — `--check`, `--checksums-only`
- `python3 lib/scenarios.py --search "keyword"` — search scenarios
